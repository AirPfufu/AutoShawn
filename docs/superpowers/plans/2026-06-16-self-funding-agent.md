# Self-Funding Agent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the first operational assets for a self-funding agent that sells B2B lead research and personalized outreach preparation.

**Architecture:** Use focused Markdown and CSV files as the first operating system. The design spec defines the business rules, operations files define daily execution, and the weekly review file defines how experiments are judged.

**Tech Stack:** Markdown, CSV, public web research, approved account workflows, HTTP/API-first integrations.

---

**Files:**
- Create: `operations/self-funding-agent/service-package.md`

- [ ] **Step 1: Write the first offer**

**Files:**

Write specific search phrases for lead generation, CRM cleanup, sales assistant, content repurposing, and AI automation demand.

### Task 4: Create Outreach Templates

**Files:**
- Create: `operations/self-funding-agent/outreach-templates.md`

- [ ] **Step 1: Write first-touch templates**

Write short templates for marketplace replies, email, LinkedIn-style messages, and website contact forms.

- [ ] **Step 2: Write follow-up templates**

Write two follow-ups that ask for a small test, not a full contract.

- [ ] **Step 3: Add compliance boundaries**

Require personalization, low frequency, honest identity, and opt-out language when appropriate.

### Task 5: Create Weekly Review System

**Files:**
- Create: `operations/self-funding-agent/weekly-review.md`

- [ ] **Step 1: Define weekly metrics**

Track sourced leads, qualified leads, approved sends, replies, calls, trials, revenue, and lessons.

Create a clear decision table for expanding profitable work, changing weak work, and stopping dead channels.

- [ ] **Step 3: Schedule weekly review**

Create a weekly automation in the current Codex thread to review the project and propose next actions.
