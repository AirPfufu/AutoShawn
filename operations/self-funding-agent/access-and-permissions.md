# Access and Permissions

## Credentials Policy

Credentials provided by the user are operational secrets.

Rules:

- Do not write passwords into repository files.
- Do not commit credentials, cookies, recovery codes, or session exports.
- Do not paste credentials into logs or public messages.
- Use accounts only for the self-funding agent project.
- Ask for explicit approval before creating an account on a new third-party platform.

## Approved Foundation

The user has provided a dedicated email account for this project. Use it only when an authorized workflow needs email access, account registration, or customer communication.

## GitHub Repository Setup

GitHub does not track empty folders. To create a project folder in a GitHub repository, create a file such as:

`self-funding-agent/README.md`

Required from user:

- Repository full name in `owner/repo` format.
- Whether to write directly to the default branch or create a branch first.

Preferred branch:

`codex/self-funding-agent-ops`

## External Platform Registration

Before creating any account, confirm:

- Platform name.
- Business purpose.
- Public-facing display name.
- Whether the platform permits agency, automation, or AI-assisted service delivery.
- Whether payment, KYC, phone verification, or captcha will require user action.
